## 🛠 Contribution Instructions & Workflow

**CRITICAL:** Do NOT push directly to the `main` or `dev` branches. Every task must have its own branch and be submitted via a Pull Request (PR).

### Step 1: Initial Setup (Koby Atilano)
Koby is responsible for the first push to set up the base repository structure.
1. Initialize the Git repository.
2. Create the base folders (`docs/`, `src/`, etc.).
3. Commit and push the initial structure to the `main` branch.
4. Create and push a `dev` branch.
5. Notify the team that the repository is ready to clone.

### Step 2: Individual Development (All Members)
When starting your assigned task, follow these steps:
1. Pull the latest `dev` branch: `git pull origin dev`
2. Create your own feature branch: `git checkout -b feature/your-feature-name` (e.g., `feature/planning-docs`, `feature/qa-testing`).
3. Complete your checklist items and commit with clear, descriptive messages.

### Step 3: Pull Requests & Merging
1. Push your feature branch to GitHub: `git push origin feature/your-feature-name`
2. Open a Pull Request (PR) against the `dev` branch.
3. Take screenshots of your PRs (required for submission).
4. Have at least one other team member review and merge the PR to resolve any conflicts collaboratively.
